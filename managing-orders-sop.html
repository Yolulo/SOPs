Orders Overview

The Orders List

Navigate to Orders → Orders in the left sidebar. The list shows all orders within the selected date range, with status filter tabs across the top.

Fig 1.1 — Orders list: All Orders tab, showing Order ID, Date & Time, Merchant, Customer, Items, Total, Type, Completed In, Payment and Status columns

List Columns Explained
Order ID
Unique sequential order number — click a row to open the order detail panel
Date & Time
When the order was placed
Merchant
Which merchant fulfilled the order — ✦ icon indicates AI Recommended
Customer
Customer name — New badge on first-time orders
Items / Total
Item count and order total value
Type
Delivery (blue dot) or Pickup (orange dot)
Completed In
✓ green tick = fulfilled · time shown is duration from order placed to completion
Payment
Payment method — e.g. Stripe, COD
Status
Current order state badge — Completed, Pending, Accepted, Ready, Collected, Cancelled
Status Filter Tabs
ALL
Every order regardless of status — count shown in brackets e.g. ALL (11)
Pending
Order placed, awaiting merchant acceptance
Accepted
Merchant has accepted the order and is preparing it
Ready
Order prepared and ready for collection or driver pickup
Collected
Driver has collected the order for delivery
Completed
Order successfully delivered or collected by customer
Cancelled
Order was cancelled at any stage
List Controls
Search Order
Search by order ID, customer name or merchant name
Per Page
Change how many orders show per page — default 25. Options typically 25, 50, 100
All Orders / Scheduled
Toggle between live orders and scheduled (pre-booked) orders
Order Details

Viewing Order Details

Click any row in the Orders list to open the order detail panel on the right. The panel has three tabs: Order, Timeline, and Customer.

Fig 2.1 — Order detail panel: Order tab selected, showing Delivery By dropdown, Pick/Drop route, order type, payment method and source

Fig 2.2 — Bill Details section: Products listed with qty/price, Subtotal, Delivery charge, Delivery Tax, and Total

Order Tab — Information Shown
Order #
Order ID and AI Recommended badge if applicable
Date & status
Order timestamp and current status badge (top right)
Delivery indicator
Blue dot = Delivery · Green dot + time = completed with duration
Delivery By
Dropdown — currently assigned delivery handler (e.g. Autozod). Can be changed here
Pick
Merchant name and pickup address (shown in blue link)
Drop
Customer name, delivery address, building/city details and mobile number
Type
Delivery or Pickup
Mop
Method of payment — e.g. Stripe
Source
Platform the order came from — e.g. ios, android, web
Bill Details
Products
Each line item — product name, quantity, and unit price
Subtotal
Product total before delivery
Delivery
Delivery fee charged to customer
Delivery Tax
Tax on delivery charge if applicable
Total
Grand total charged to customer
Other Tabs
Timeline tab
Shows a timestamped log of every status change — useful for investigating delayed or disputed orders
Customer tab
Quick view of the customer profile linked to this order
Order Management

Changing Order Status & Delivery Provider

Admins can manually override both order status and delivery provider directly from the order detail panel.

Fig 3.1 — Status dropdown open on order #1079: options are Pending, Accepted, Ready, Collected, Completed, Cancelled

Changing Order Status
How to change
Open the order detail panel → click the green status badge (top right, e.g. Completed ▼) → select the new status from the dropdown
Pending
Revert to pending — use if a merchant accidentally accepted or if order needs re-review
Accepted
Mark as accepted by merchant
Ready
Mark as prepared and ready for driver or customer collection
Collected
Mark as collected by driver
Completed
Mark as successfully delivered or collected
Cancelled
Cancel the order — use with care, communicate with merchant and customer
Changing Delivery Provider
Delivery By dropdown
In the order detail panel → click the Delivery By dropdown → select the provider. Green tick = currently confirmed provider
Autozod
Default automatic driver assignment — standard for Yolulo deliveries
⚠️
Status changes take effect immediately and are visible to the merchant and customer. Only change status if you are certain — reversing completed or cancelled orders may cause confusion.
Order Notes

Adding an Order Note

Internal notes can be attached to any order for team reference — visible to admin only, not to the merchant or customer.

Fig 4.1 — Add Order Note modal for Order ID 1079: free-text area with Cancel and Update buttons

Adding a Note
Access
Orders list → find the order row → click the 📋 note icon (if visible) or open the order detail panel and look for the note/comment action
Order ID
Shown at top of modal for confirmation — verify you have the right order
Note text
Free text — write anything relevant: delivery issue, customer complaint, refund reason, driver note, etc.
Update
Click Update to save the note to the order record
💡
Use order notes to log any manual interventions, customer calls, or issues so the full team has visibility without having to check external systems.
Filtering

Filtering Orders by Date

Use the date range picker to view orders for a specific period. The current range is shown in the top right of the Orders page.

Fig 5.1 — Date range picker: quick presets on the left (Today, Yesterday, This Month, This Year, Last Month, Last 30 Days) and dual calendar for custom range

Date Filter Options
Access
Click the 📅 date range shown in the top right of the Orders page (e.g. "11 February 2026 – 13 March 2026 ▼")
Quick presets
Today · Yesterday · This Month · This Year · Last Month · Last 30 Days — click to apply instantly
Custom range
Click a start date on the left calendar, then an end date on the right calendar. Selected range is highlighted in blue
Apply
Click the green Apply button to load orders for the selected range
Cancel
Click Cancel to dismiss without changing the range
Scheduled Orders

Viewing Scheduled Orders

Scheduled orders are pre-booked by customers for a future time slot.

Accessing Scheduled Orders
Toggle
On the Orders page, click the Scheduled tab (next to All Orders) to switch to the scheduled orders view
Columns
Same columns as All Orders — Order ID, Date & Time (showing the scheduled slot), Merchant, Customer, Total, Type, Payment, Status
Managing
Scheduled orders go through the same status flow as regular orders — the merchant needs to accept them before the scheduled time
ℹ️
Scheduled orders will remain in Pending status until the merchant accepts them. Monitor these to ensure merchants are not missing pre-booked slots.
Cart Recovery

Viewing & Actioning Cart Recovery

Cart Recovery (Beta) shows customers who added items to their cart but did not complete checkout. Use this data to identify re-engagement opportunities.

Fig 7.1 — Cart Recovery list: Total Abandoned count (4), with table showing Abandoned At, Merchant, Customer, Items and Total per row

Fig 7.2 — Cart Recovery detail panel: User ID, Name, Email, Phone, and item breakdown (Product, Quantity, Total) with Cart Total

Cart Recovery Overview
Access
Orders → Cart Recovery in the left sidebar sub-menu
Total Abandoned
Summary card showing how many carts have been abandoned in the selected date range
Table columns
Abandoned At · Merchant · Customer · Items · Total
Date range
Use the date picker (top right) to filter the time period — same control as on the Orders page
Viewing Cart Details
Click a row
Opens the Cart Recovery detail panel on the right
Customer info
User ID, Name, Email and Phone number — can be used to follow up directly with the customer
Cart contents
Each abandoned product with image, name, quantity and line total
Cart Total
Total value of the abandoned cart — useful for prioritising high-value recovery
Abandoned At / Created At
Shown at the top of the panel — the time the cart was abandoned vs when it was first created
💡
Cart Recovery is currently in Beta. Use the customer contact details to reach out via phone or email for high-value abandoned carts. Future versions may include automated re-engagement messaging.
