Overview

WebHooks Page

Webhooks let Autozod push real-time event notifications to an external URL whenever a task is assigned or its status changes. This is how Autozod communicates task updates back to Hyperzod.

💡
Navigation: Autozod top nav → Webhooks
WebHooks page — Create WebHooks form (top), current webhooks list (middle), and WebHook Log (bottom).
Creating

Create a Webhook

Add a new webhook to notify an external endpoint when a specific Autozod event occurs.

Create WebHooks form — Choose Event dropdown and URL field. Current webhooks listed below (Task Assigned, Task Status Updated).
Create WebHooks Form
Choose Event Required
Select the event to listen for from the dropdown. Available events: Task Assigned — fires when a driver is assigned an order. Task Status Updated — fires on every status change (CREATED, AWAITING, ASSIGNED, PICKED_UP, COMPLETED, etc.).
URL Required
The endpoint URL that should receive the POST payload when the event fires. Example: https://delivery.apps.hyperzod.com/api/v1/6199/webhook/order/autozod
CREATE
Click to save the webhook. It becomes active immediately and appears in the Current Webhooks list below.
🔗
Hyperzod webhook URL format: https://delivery.apps.hyperzod.com/api/v1/[TEAM_ID]/webhook/order/autozod — replace [TEAM_ID] with your Hyperzod store/team ID. Both Task Assigned and Task Status Updated webhooks should point to this URL.
Managing

Current Webhooks

View and delete existing webhooks. The current webhooks list shows all active event listeners.

Current Webhooks — Event · URL · Delete columns. Two webhooks shown: Task Assigned and Task Status Updated, both pointing to the Hyperzod endpoint.
Webhooks Table
Event
The Autozod event this webhook listens for (Task Assigned or Task Status Updated).
URL
The destination endpoint receiving the webhook payload.
Delete (🗑️)
Click the bin icon to permanently delete the webhook. It stops firing immediately. There is no confirmation step.
⚠️
Deleting a webhook stops all event notifications to that URL. If you delete the Hyperzod webhook, Autozod will no longer update order statuses in the Hyperzod platform. Re-create it immediately if deleted by mistake.
Debugging

WebHook Log

The WebHook Log is a chronological history of every event fired. Use it to confirm webhooks are firing correctly or to debug integration issues.

WebHook Log — columns: Event · Date · URL · Data (eye icon). Each row is one webhook fire. Click Reload to refresh the log.
Log Columns
Event
The event type that triggered the webhook (e.g. TaskStatusUpdated, TaskAssigned).
Date
Exact timestamp the webhook fired (e.g. March 12th 2026, 10:39:10 pm).
URL
The destination URL the payload was sent to.
Data (👁)
Click the eye icon to expand the full JSON payload sent with that webhook event.
Reload
Click the Reload button to refresh the log with the latest entries.
Expanded webhook payload — the full JSON body sent to the endpoint, including task ID, status, order ID, merchant ID, pickup/drop objects, driver commission, and proof requirements.
Payload Fields (TaskStatusUpdated)
_id
Autozod Task UUID.
status
Current task status (e.g. COMPLETED, ASSIGNED, PICKED_UP).
order_id
The Hyperzod order number this task relates to.
merchant_id
The Hyperzod merchant UUID for the pickup location.
pickup / drop
Expandable objects containing address, contact, and timestamp data for pickup and drop-off points.
is_task_ready
true when the task is ready for assignment.
is_pickup_proof_required
true when the driver must take a photo at pickup.
🔍
Debugging tip: If order statuses aren't updating in Hyperzod, check the Webhook Log first. If TaskStatusUpdated events are appearing in the log, Autozod is firing correctly — the issue is on the Hyperzod receiving end. If no events appear, check the webhook URL is correct and the webhook exists in the Current Webhooks list.
