Overview

The Customers List

Click Customers in the left sidebar to open the customer management area. The list has two tabs: Customers and Segmentation.

Fig 1.1 — Customers list: Name, Email, Phone, Sign Up Date, Last Login, Status (Active badge), Actions (edit/delete icons) and Auto Login button per row

List Columns
Name
Customer display name
Email
Registered email address
Phone
Mobile number with country code
Sign Up Date
Date and time the customer registered
Last Login
Relative time since last app/web login — e.g. "5 days ago"
Status
Active = account in good standing · Blocked = account suspended
List Controls & Actions
Search Customers
Search by name, email or phone number
All / Active / Block tabs
Filter customers by account status — All shows everyone, Active shows only enabled accounts, Block shows suspended accounts
✏️ Edit icon
Opens the Edit Customer modal — update name, phone or tags
🗑️ Delete icon
Permanently deletes the customer account — requires confirmation
Login button (Auto Login)
Log into the storefront as that customer — useful for troubleshooting customer-reported issues
Import / Export
Bulk import customers from CSV or export all customer data — buttons in the top right
+ Add Customer
Manually create a new customer account
Adding Customers

Adding a New Customer Manually

Use this to create a customer account on behalf of a user — for example, when onboarding a corporate account or correcting a registration issue.

Fig 2.1 — Add Customer modal: First name (required), Last name, Email Address (required), Mobile Number (required), Tags

Add Customer Fields
First name Required
Customer's first name
Last name
Optional surname
Email Address Required
Must be a valid unique email address — this becomes their login
Mobile Number Required
Phone number with country code — select flag for country prefix
Tags
Optional — add searchable tags to segment or categorise this customer (e.g. VIP, Corporate, Staff)
Save
Creates the account immediately. The customer will need to set a password via the app/storefront login
Edit & Delete

Editing & Deleting a Customer

Fig 3.1 — Edit Customer modal: same fields as Add Customer, pre-populated — Email is greyed out (read-only)

Editing
Access
Customers list → click ✏️ edit icon on the customer row
Editable fields
First name, Last name, Mobile Number, Tags
Email
Read-only — email address cannot be changed once set
Save
Click Save to apply changes

Fig 3.2 — Delete Customer confirmation modal: "Are you sure you want to delete Customer Dan?" with Cancel and Delete (red) buttons

Deleting
Access
Customers list → click 🗑️ delete icon on the customer row
Confirmation
A modal asks "Are you sure you want to delete Customer [name]?" — review the name carefully before confirming
Delete button
Red — click to permanently delete the account and all associated data
Cancel
Dismisses without deleting
⚠️
Deletion is permanent. All customer data including order history, reviews and saved addresses will be removed. Consider blocking the account instead (via Status) if there is any chance the data may be needed.

Profile Panel Contents

Avatar, customer name, Contact number, Email, Sign Up Date, Total orders count
Orders tab

Reviews tab
Any reviews the customer has left for merchants
Logging In as a Customer
Purpose
Use to reproduce a customer-reported issue, verify their experience, or troubleshoot checkout/ordering problems
Access
Customers list → find the customer → click the green Login → button (Auto Login column)
Effect
Opens the storefront logged in as that customer. Any actions taken will be on their account — do not place test orders
⚠️
Login as customer with care. You are accessing their live account. Do not modify orders, addresses or payment details unless explicitly requested by the customer.
Bulk Operations

Importing & Exporting Customers

Both tools are accessible via the Import and Export buttons in the top right of the Customers page.

Import Customers
Step 1
Click Import (top right of Customers page) to open the Customers Import page
Step 2 — Template
Click the blue Download link to get the reference CSV template — always use this
Step 3 — Populate
Fill in customer data following the exact column structure of the template. Column mismatches will cause failed or incorrect imports
Step 4 — Upload
Drag and drop your completed CSV into the upload area, or click Browse
Step 5 — Monitor
The import history table shows Date, Status, Import ID and a Download Import File link. Refresh to check progress
Export Customers
Step 1
Click Export (top right of Customers page) to open the Customer Export page
Step 2
Click the Export button to queue the export job
Step 3
Once complete, the history table shows the export with a Download link — click to save the CSV
ℹ️
The Customer Import/Export pages are identical in layout to the Merchant Import/Export. The same rules apply — always use the reference template for imports, and never modify column headers.
Segmentation

Customer Segments

Segments are dynamic customer groups built on filter conditions. They auto-update as customer data changes — no manual maintenance required.

Fig 6.1 — Customers: Segmentation tab selected, showing existing segments list (currently empty)

Fig 6.2 — Create Segment form: Name field, Filters section with + Add Condition button, and Preview table

Fig 6.3 — Create Segment with two conditions joined by AND: "Total Number Of Orders is greater than [value]" conditions

Creating a Segment
Navigate
Customers → Segmentation tab → click + Create New (or equivalent create button)
Name
Give the segment a clear descriptive name — e.g. VIP Customers, Inactive 30 Days, High Spenders
Add Condition
Click + Add Condition to add a filter row
Condition structure
Each row has three parts: [Field] + [Operator] + [Value] — e.g. "Total Number Of Orders | is greater than | 5"
AND logic
Click + Add Condition again to add more conditions — multiple conditions are joined with AND (all must be true)
Preview
The Preview table at the bottom updates in real time to show which customers currently match the conditions
Save
Click + Save (top right) to create the segment. It will appear in the Segmentation tab
Available Filter Fields (examples)
Total Number Of Orders
Filter by how many orders a customer has placed — use "is greater than", "is less than", "equals"
Other fields
The field dropdown may include: Sign Up Date, Last Order Date, Total Spend, City, Tags — explore the dropdown for all options
💡
Segments auto-update. Once created, the segment membership refreshes automatically as customer data changes — e.g. a "5+ orders" segment will add customers as they hit that threshold without any manual action.
