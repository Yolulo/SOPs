Getting Started

Opening the Edit Merchant Form

All merchant settings are managed from a single Edit Merchant page. Navigate there first, then jump to the relevant section below.

Fig 1.1 — Merchants list: click the ✏️ edit icon in the Action column to open Edit Merchant

Navigation Path
Step 1
Click Merchants in the left sidebar
Step 2
Find the merchant in the list — use Search Merchants if needed
Step 3
Click the ✏️ edit icon in the Action column to open the Edit Merchant form
Save
Click the blue Save button (top right) after making any changes. Changes do not auto-save
ℹ️
Merchants list columns: Publish = storefront visibility toggle · Online = green dot when merchant is currently accepting orders · Login = enter the merchant's own admin panel

Section — Category

Editing Merchant Category

The Category field controls which business category tag is shown on the merchant's storefront listing.

Category Field
Current category
Shown as a dark pill tag (e.g. South American ×) — click × to remove
Change category
Click × to remove the current one, then search and select the new category from the dropdown
Multiple categories
A merchant can be assigned to more than one category — select additional ones from the dropdown
💡
Category options come from the Categories list (Merchants → Category tab). If the category you need doesn't exist, create it there first before coming back here.

Editing Order Types & Default Order Type

Controls which order types customers can select, and which is pre-selected by default.

Order Types Required

Section — Status & Visibility

Status, Publishing & Online Indicator

Three separate controls govern whether a merchant is live — understand the difference between all three.

Fig 5.1 — Merchants list: Publish toggle (storefront visibility) and Online dot (live order acceptance indicator)

The Three Visibility Controls
Status (Edit Merchant form)
Active = merchant is configured and ready · Inactive = hidden regardless of Publish toggle. Change via the Status dropdown in the Edit Merchant form → Save
Publish (Merchants list)
Blue toggle = merchant is visible on the storefront. Grey = hidden from customers even if Status is Active. Toggle directly from the Merchants list — no Save needed
Online dot (Merchants list)
Green dot = merchant is currently accepting orders in real time. This is controlled by the merchant themselves from their own panel — admin cannot set this directly
⚠️
All three must align for a merchant to be live: Status = Active AND Publish = on AND merchant has toggled themselves Online. A merchant can be Active and Published but show no green dot if they haven't opened their panel.
Section — Delivery

Delivery By, Delivery Provider & Zones

Controls who fulfils delivery and which provider handles driver assignment.

Delivery Configuration
Delivery By
Admin — Yolulo manages delivery using the platform's driver pool (standard Yolulo model)
Merchant — the merchant handles their own delivery directly
Delivery Provider
Autozod — automatic driver assignment system. Only change this if moving to a different dispatch method
Zones
Assign or change the delivery zone this merchant operates within
Share Customer Detail
Toggle on to pass customer contact details to the merchant (useful when Delivery By = Merchant)
⚠️
Changing Delivery By to Merchant means the merchant is responsible for all deliveries. Ensure the merchant is set up to handle this before making the change — Autozod will no longer dispatch drivers for their orders.
Section — Commission

Editing Commission Rates

Commission is set independently per order type. Click each tab to view and update that type's rate.

Fig 7.1 — Commission: Delivery tab selected, 10% commission + £0 fixed

Fig 7.2 — Commission: Pickup tab selected, 0% commission

Fig 7.3 — Commission: Custom tab selected — set independently of other order types

Commission Fields
Order type tabs
Click Delivery / Pickup / Custom / Custom 2 to set rates per type
Percentage (%)
Commission as a percentage of the order value — e.g. 10 = 10% of each order
Fixed (£)
Flat fee per order in addition to (or instead of) the percentage. Both can be combined
💡
Example: Delivery set to 10% + £0 means Yolulo takes 10% of every delivery order. Pickup set to 0% means no commission is charged on collection orders.
Section — Taxes

Editing Tax Settings

Controls whether VAT is shown as included in product and delivery prices.

Fig 8.1 — Taxes section: Include tax in prices checkbox — when checked, prices are treated as VAT-inclusive

Tax Configuration
Include tax in prices
Checked — product prices and delivery charges already include VAT. No tax is added on top at checkout
Unchecked — prices are ex-VAT; tax may be added at checkout depending on product-level tax settings
⚠️
Confirm with the merchant whether their prices are VAT-inclusive before changing this setting. Incorrect tax configuration affects what customers are charged at checkout.
Section — Custom Text

Editing the Custom Text Banner

A short promotional or informational message displayed on the merchant's storefront page.

Custom Message
Custom Message
Up to 40 characters — shown as a banner on the storefront. Example: South American Street Food!
Language
English (default) — update if a different language is needed
💡
Keep it punchy — 40 characters is short. Good examples: "Fresh handmade pasta daily" · "Order before 9pm for same night" · "Halal certified kitchen"
Section — Timezone

Editing the Merchant Timezone

The timezone controls how store timings and scheduled orders are interpreted for this merchant.

Fig 10.1 — Edit Merchant bottom section: Checkout Custom Fields, Timezone (Europe/London), Payment Settings, Delete

Timezone Setting
Timezone

Section — Payment Settings

Editing Payment Methods

Allows you to override the platform-wide payment methods for this specific merchant.

Opening Payment Settings
Navigate to
Edit Merchant → scroll to Payment Settings section → click ⚙ Settings

Fig 11.1 — Payment Methods: manual methods (COD, Bank Deposit, Custom PG) and online methods (Stripe enabled, plus 12 other providers)

Payment Method Options
Override manual payment methods
Check to customise manual payment options (COD, Bank Deposit, Custom PG) for this merchant specifically
COD
Cash on delivery — toggle to enable/disable for this merchant
Bank Deposit
Manual bank transfer payment option
Override online payment methods
Check to customise which online gateways are available for this merchant
Stripe Enabled
Standard payment gateway for the Yolulo model — should remain enabled
Other providers
PayPal, Coinbase, Paystack, PayU India, Multisafepay, Wipay, Mollie, Paytm, Ipay88, Mercadopago, Squareup, and more — enable only if required for this merchant
⚠️
Only override payment methods if there is a specific reason for this merchant to differ from the platform default. For standard Yolulo merchants, Stripe should be the active online method.
Section — Merchant Slug

Editing the Merchant Slug (URL)

The slug is used in the merchant's storefront URL. It auto-generates from the merchant name but can be edited.

Slug Field
Slug Required
URL-safe identifier — e.g. chamo-street-food-1. Lowercase, hyphens only, no spaces
⚠️
Changing the slug changes the merchant's URL. Any existing links or QR codes pointing to the old URL will break. Only change if absolutely necessary and communicate the new URL to the merchant.
Section — Account Deletion

Deleting a Merchant

Permanent, irreversible removal of the merchant account and all associated data.

Delete Merchant
Location
Edit Merchant → scroll to the bottom → Merchant Account Deletion section → click 🗑 Delete Merchant (red link)
What is deleted
The merchant account and all associated data — products, orders, settings, users, payout configuration
⚠️
This action is permanent and cannot be undone. Before deleting: confirm all outstanding orders are completed, all payouts are settled, and the merchant has been informed. Consider setting Status to Inactive and Publish to off instead of deleting if there is any chance of reactivation.
